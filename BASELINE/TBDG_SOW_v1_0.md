## 1. วัตถุประสงค์ของการจัดทำ Statement of Work

เอกสารนี้จัดทำขึ้นเพื่อกำหนดขอบเขตและรายละเอียดของการพัฒนา **TradeBridge Digital** โดยมีวัตถุประสงค์เพื่อเป็นแนวทางในการพัฒนาและส่งมอบระบบให้แก่ผู้ใช้งานที่เกี่ยวข้อง อธิบายขอบเขตหน้าที่ของระบบในแต่ละโมดูล รวมถึงระยะเวลา งานส่งมอบ และงวดการชำระเงินของโครงการ

---

## 2. วัตถุประสงค์ของโครงการ

* พัฒนาระบบที่ช่วยให้ผู้ประกอบการนำเข้าสามารถวิเคราะห์ต้นทุนโลจิสติกส์และภาษีที่เกี่ยวข้องก่อนสั่งซื้อ (Pre-trade / Cost Estimation)
* ให้ระบบแนะนำพิกัดศุลกากร (Tariff Intelligence) จากข้อมูลและรูปภาพสินค้า เพื่อลดความเสี่ยงด้าน classification
* เตรียมความพร้อมในการจัดทำใบขนสินค้าขาเข้า (Customs Entry Preparation) และตรวจสอบความครบถ้วนของเอกสารก่อนยื่น
* รองรับการตรวจสอบความถูกต้องของการนำเข้าย้อนหลัง (Post-Audit / Import Entry Verification)
* ส่งมอบระบบในรูปแบบเว็บแอปพลิเคชันแบบ responsive ภายใต้ขอบเขต MVP ที่ตกลงร่วมกัน

---

## 3. ขอบเขตของโครงการ

การพัฒนาและจัดการระบบจะครอบคลุมโมดูลหลัก ดังนี้:

### 3.1 Module 1 — การวิเคราะห์ต้นทุนโลจิสติกส์ของการนำเข้า (Logistics Import Cost Estimation)

* **3.1.1 รับข้อมูลนำเข้า**: สินค้า ประเทศต้นทาง/ปลายทางและท่า Incoterm ถิ่นกำเนิด รูปแบบขนส่ง พิกัดศุลกากรระดับ 8 หลักพร้อมรหัสสถิติ ราคาสินค้า ค่าขนส่ง และค่าประกันภัย
* **3.1.2 คำนวณราคา CIF และภาษีที่เกี่ยวข้อง**: ภาษีศุลกากร VAT ภาษีสรรพสามิต (ถ้ามี) และภาษีเพื่อมหาดไทย (ถ้ามี)
* **3.1.3 ประมาณค่าใช้จ่ายโลจิสติกส์ปลายทาง**: ค่าเอกสาร ค่าใช้จ่ายภายในท่า/สนามบิน และค่าบริการที่เกี่ยวข้อง
* **3.1.4 แนะนำข้อกำหนดการนำเข้า (OGA / Permit)** และสิทธิประโยชน์ FTA (MVP: ACFTA, AFTA/ATIGA, RCEP) พร้อมประมาณการประหยัดภาษี
* **3.1.5 ประมาณระยะเวลาการขนส่ง** และแนะนำผู้ให้บริการขนส่ง/ตัวแทนออกของที่รับอนุญาต

### 3.2 Module 2 — การวิเคราะห์สินค้านำเข้าเพื่อแนะนำพิกัดศุลกากร (Tariff Intelligence)

* **3.2.1 รับข้อมูลสินค้าเพื่อตีพิกัด**: รูปภาพสินค้า ข้อมูลทางเทคนิค พิกัดจากประเทศต้นทาง และพิกัดที่คาดว่าจะเป็น (ถ้ามี)
* **3.2.2 แนะนำพิกัดศุลกากรระดับ 6–8 หลัก** พร้อมระดับความมั่นใจ (confidence)
* **3.2.3 Tariff Intelligence DB (MVP)**: HS Code ยุทธศาสตร์ 200 รายการ และอัตราภาษี MFN + ACFTA + AFTA + RCEP

### 3.3 Module 3 — การเตรียมความพร้อมการจัดทำใบขนสินค้าขาเข้า (Customs Entry Preparation)

* **3.3.1 รับเอกสารประกอบการนำเข้า**: Commercial Invoice, Packing List, ใบตราส่ง, เอกสารราคาศุลกากร, ข้อมูลผู้นำเข้า/เลขผู้เสียภาษี และเอกสารสิทธิประโยชน์
* **3.3.2 สกัดและจัดเตรียมข้อมูลใบขน** ให้ครอบคลุมฟิลด์สำคัญสำหรับการยื่นใบขนสินค้าขาเข้า
* **3.3.3 Pre-flight verification**: ตรวจสอบความครบถ้วนและความสอดคล้องของเอกสารก่อนยื่น พร้อม foreshadow ต้นทุนภาษี
* **3.3.4 จัดเตรียมข้อมูลเพื่อส่งต่อไปยังซอฟต์แวร์/NSP** (data passthrough) เพื่อลดการกรอกข้อมูลซ้ำ

### 3.4 Module 4 — การตรวจสอบความถูกต้องในการนำเข้าย้อนหลัง (Post-Audit / Import Entry Verification)

* **3.4.1 รับใบขนและเอกสารประกอบ** เพื่อตรวจสอบย้อนหลัง
* **3.4.2 แสดงความคลาดเคลื่อนหรือผิดพลาด** ในการสำแดง (เช่น การตีพิกัด การสำแดงราคา การใช้สิทธิ FTA ความครบถ้วนของเอกสาร)
* **3.4.3 ให้ข้อแนะนำเบื้องต้น** สำหรับผู้ประกอบการในการดำเนินการด้านศุลกากรเมื่อพบความผิดพลาด
* **3.4.4 จัดเก็บผลการตรวจสอบ** เพื่ออ้างอิงและติดตาม

### 3.5 ขอบเขตนอกจากการพัฒนาในรอบนี้ (Out of Scope / Phase ถัดไป)

* Mobile Native Application
* ความตกลง FTA เพิ่มเติมนอกเหนือจาก ACFTA, AFTA/ATIGA และ RCEP
* ML Model / CBAM Engine ขั้นสูง

## 4. ระยะเวลาในการดำเนินโครงการ

ระยะเวลาของโครงการนี้ใช้เวลาประมาณ **4 เดือน (16 สัปดาห์)** ตั้งแต่เดือน **กรกฎาคม – ตุลาคม พ.ศ. 2569** โดยเริ่มนับจากวันที่เริ่มโครงการอย่างเป็นทางการ และจะมีการติดตามและรายงานความคืบหน้าของโครงการเป็นประจำ

---

## 5. งวดการชำระเงิน

ค่าพัฒนาซอฟต์แวร์และระบบที่เกี่ยวข้องที่ลูกค้าจ่ายให้ผู้พัฒนาสำหรับขอบเขต MVP รวมทั้งสิ้น **1,999,900 บาท**  
โครงการนี้จะมีงวดการชำระเงินแบ่งเป็น **3 งวด (30% / 50% / 20%)** ดังนี้:

### งวดที่ 1 (30%) — **599,970 บาท**

ส่งมอบเอกสาร ภายในวันที่ **15 สิงหาคม พ.ศ. 2569**

* แผนการดำเนินโครงการ (PMP)
* เอกสารสรุปความต้องการของระบบงาน (SRS)
* เอกสารการวิเคราะห์และออกแบบระบบ (SDD)

### งวดที่ 2 (50%) — **999,950 บาท**

หลังจากเสร็จสิ้นการพัฒนาระบบหลักและฟังก์ชันสำคัญตามขอบเขต MVP (Module 1–4 ตามที่ระบุในข้อ 3) จะส่งมอบ ภายในวันที่ **30 กันยายน พ.ศ. 2569**

* ซอฟต์แวร์ระบบ (Software Components / Web Application)

### งวดที่ 3 (20%) — **399,980 บาท**

หลังจากทดสอบระบบ (User Acceptance Test) และส่งมอบระบบพร้อมใช้งาน จะส่งมอบ ภายในวันที่ **31 ตุลาคม พ.ศ. 2569**

* เอกสาร User Acceptance (UAT)
* เอกสาร Product Operation Guide (POG)
* เอกสาร Maintenance Document (MA)
* เอกสาร Software User Document (SUD)
* การอบรมผู้ดูแลระบบและผู้ใช้งาน ระยะเวลา 2 วัน

| งวด | สัดส่วน | จำนวนเงิน (บาท) | กำหนดส่งมอบโดยประมาณ |
| --- | ------: | --------------: | -------------------- |
| 1 | 30% | 599,970 | 15 ส.ค. 2569 |
| 2 | 50% | 999,950 | 30 ก.ย. 2569 |
| 3 | 20% | 399,980 | 31 ต.ค. 2569 |
| **รวม** | **100%** | **1,999,900** | |

## 6. การรายงานความคืบหน้าของโครงการ

ผู้จัดการโครงการจะจัดส่งรายงานความคืบหน้าให้แก่ผู้เกี่ยวข้องทุก ๆ 1 เดือน เพื่อให้มั่นใจว่าโครงการดำเนินไปตามแผน ครอบคลุมความคืบหน้าของงาน ปัญหาอุปสรรค การปรับแผน (ถ้ามี) และความเสี่ยงที่เกี่ยวข้อง

---

## 7. ปัจจัยที่มีผลกระทบต่อโครงการ

* ความพร้อมของผู้ใช้งานในการให้ข้อมูลที่ถูกต้องและครบถ้วน
* ความถูกต้องและความทันสมัยของข้อมูลอัตราภาษี / สิทธิประโยชน์ FTA
* การเปลี่ยนแปลงนโยบายหรือกฎระเบียบด้านศุลกากรที่เกี่ยวข้อง
* ความพร้อมและการเชื่อมต่อกับระบบภายนอก (เช่น NSP / ซอฟต์แวร์ยื่นใบขน) ที่อยู่นอกการควบคุมโดยตรงของทีมพัฒนา
* ความเสถียรของเทคโนโลยีและโครงสร้างพื้นฐานที่ใช้ในการพัฒนาและใช้งานระบบ

---

## 8. การบริหารการเปลี่ยนแปลง

การเปลี่ยนแปลงหรือการเพิ่มเติมขอบเขตของโครงการจะได้รับการพิจารณาและอนุมัติผ่านกระบวนการบริหารจัดการความต้องการ (Change Request) เพื่อให้สอดคล้องกับเป้าหมาย ระยะเวลา และค่าพัฒนาของโครงการ

---

## 9. ข้อตกลงร่วมกัน

ทุกฝ่ายจะต้องปฏิบัติตามข้อตกลงและแผนการดำเนินงานตามที่ระบุในเอกสารนี้ เพื่อให้โครงการสามารถดำเนินไปอย่างมีประสิทธิภาพและบรรลุวัตถุประสงค์ที่กำหนด
